## Рубрика

**CRITICAL — сломан шов конституции** (компаундится между итерациями, ведёт к болезненному рефактору или убивает смысл проекта):
- **Пробитая A2A-граница:** planner импортирует код supplier (или наоборот); появился общий модуль схем (`shared/`) — схемы дублируются осознанно (DECISIONS); planner знает supplier не через `SUPPLIER_CARD_URL` из env (хардкод host/port в коде); оба агента слиты в один процесс/раннер там, где SPEC требует два.
- **Борьба с фреймворком вместо демонстрации владения им:** ручной A2A-клиент (`a2a_client.py`, свои HTTP-вызовы) вместо `RemoteA2aAgent`; ручная сборка/раздача agent card вместо `to_a2a()` / `adk api_server --a2a`; свой UI/фронтенд вместо `adk web`; самодельный оркестратор там, где есть `SequentialAgent`/`ParallelAgent`.
- **Сломан $0-контракт:** Pro-модель или любая платная модель (free tier only — Flash/Flash-Lite); плавающий алиас модели (`-latest`) или model id мимо `.env`; реальный HTTP/скрапинг/live-API в стабах (`search_flights`/`search_hotels` обязаны быть фикстурами); недетерминированный рандом в стабах.
- **Дрейф пинов:** зависимость без пина в `pyproject.toml`; апгрейд `google-adk`/`a2a-sdk` посреди проекта без записанного блокера (CLAUDE.md: пиннуем newest stable на спайке, дальше не трогаем).
- **Тест или CI-путь, способный уйти в живой LLM-вызов или сеть** (CLAUDE.md: tests never call the model).
- Закоммичен секрет (`.env` с реальным ключом, хардкод ключа).
- **Разрастание домена сверх фикстуры:** новые поля контрактов сверх SPEC, третий доменный концепт, третий суб-агент, «обогащение» стабов логикой — главный failure mode siblings, явно запрещён.
- Нарушение решённой развилки из DECISIONS без её пересмотра: merge стал LLM-шагом; model map перекроен; state-ключи фан-аута не `flights`/`hotels`.

**BUG — нарушен контракт:** структура `TripRequest`/`SupplierOffer` расходится между сторонами (дубль обязан оставаться синхронным — дрейф ловится валидацией, но заведомо рассинхроненные схемы — баг); суб-агент пишет не в свой state-ключ / ключи конфликтуют; `parse` совмещает tools с `output_schema` (ADK это запрещает); агент-пакет не дискаверится в `adk web` (нет `__init__.py` с `root_agent`); env-чтение раскидано по коду вместо одного места.

**NIT — отклонение от аналога/именования**, судейский вызов. Только с конкретным аналогом.

**Срез без маркера (NIT).** Сознательный угол (упрощение, заглушка, «одного хватит») без комментария — отметь как NIT с предложением добавить маркер `# tw-lite: <потолок> → <апгрейд>`. Срез *с* маркером — не находка: это штатный режим проекта. Не превращай маркер в повод докрутить срез до «по-взрослому».

## Что НЕ флагаешь (existence-gate — иначе шум)

Качество LLM-выхода (хорошо ли parse парсит, красив ли план) · реализм данных в стабах (цены, названия отелей) · качество промптов · edge-cases toy-домена · прод-хардненинг (auth, персистентность, retries, rate-limit, observability/eval control planes — территория siblings, деплой) · security сверх утечки секретов · широта тестов. Всё это проект осознанно режет (CLAUDE.md → Non-goals).

**Но красную линию — флагаешь.** Existence-gate режет реализм и точность, НЕ: корректность демонстрируемой техники, честность A2A-границы, направление зависимостей, утечку секретов, $0-контракт, продвижение красной нити. Кейсы-нарушения (CRITICAL/BUG по тяжести):
- **A2A, которого нет:** «протокольный вызов» на деле импорт/локальный вызов; хоп недоказуем (не виден в логе supplier-процесса).
- **«Параллельный» фан-аут, который на деле последовательный** — иерархия-дифференциатор превращается в декорацию.
- **Демо, завязанное на свой UI/скрипт** вместо `adk web`/`adk run` — тулинг-дифференциатор не демонстрируется.
- **Итерация не двигает заявленный дифференциатор** — красная нить из ROADMAP не стала демонстрируемой.
