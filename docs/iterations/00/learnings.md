# Learnings 00 — что спайк рассказал про ADK 2.5.0

## Решения и почему

- **Пины сняты с «самого свежего stable» в день спайка** (2026-07-22):
  `google-adk[a2a]==2.5.0`, `a2a-sdk[http-server]==1.1.2`, `pydantic==2.13.4`,
  Python 3.13 (совпадает с dossier-lite, самый свежий из проверенных у
  сиблингов). Дальше версии заморожены: A2A-поверхность ADK experimental, и
  любой апгрейд — это лотерея breaking changes; играть в неё посреди проекта
  незачем.
- **`to_a2a()` + uvicorn, а не `adk api_server --a2a`.** Оба пути — «родные»
  для фреймворка, но CLI монтирует карты под пер-агентным префиксом
  (`/a2a/<agent_name>/...`), а `to_a2a()` отдаёт карту на корневом
  `/.well-known/agent-card.json` — ровно там, куда указывает `SUPPLIER_CARD_URL`
  и где её ждёт каноничный well-known-паттерн. Решение записано в DECISIONS.md.
- **Модели: `gemini-3.6-flash` / `gemini-3.5-flash-lite`.** Конкретные stable
  ids (не `-latest`, не Pro), живут только в `.env`; смок-тест
  `test_model_ids_are_concrete_free_tier` регрессионно охраняет оба правила.
- **Тест пинов читает `pyproject.toml` через `tomllib`,** а не дублирует версии
  литералами: источник истины один, тест сверяет «установленное == заявленное».

## Грабли (где ADK повёл себя не как ожидалось)

- **`google-adk[a2a]` не самодостаточен.** Сервер, собранный `to_a2a()`, падает
  на старте с `ModuleNotFoundError: sse_starlette`: серверные роуты `a2a-sdk`
  живут за его собственным extra `http-server`, который ADK-шный extra `a2a` не
  транзитит. В LangGraph-мире обвязку пишешь сам и зависимости видишь; здесь об
  этом рассказал только падающий процесс. Фикс — `a2a-sdk[http-server]==1.1.2`
  в pyproject.
- **Вся A2A-поверхность помечена EXPERIMENTAL** и предупреждает при каждом
  импорте (`to_a2a`, `RemoteA2aAgent`, конвертеры). Это не шум в нашем коде, а
  статус фреймворка: ещё один аргумент за жёсткую заморозку версий. В pytest эти
  предупреждения видны как `9 warnings` — гасить их фильтрами не стали, они
  честно напоминают, на чём стоим.
- **Формат карты уехал от туториалов.** В a2a-sdk 1.x у карты нет верхнеуровневых
  `url`/`protocolVersion` — эндпоинт живёт внутри `supportedInterfaces`. Всё, что
  написано про карты для a2a-sdk 0.x, читать с поправкой.
- **Ошибка модели round-trip-ится через границу как обычный ответ.** Прогон без
  валидного ключа дал на стороне caller-а `[hello_remote]: 400 INVALID_ARGUMENT
  (API key not valid)` — то есть A2A-канал доказуем даже без работающего LLM:
  `GET` карты и `POST /` в логе сервиса были, а упало уже внутри агента. Удобно
  для отладки границы отдельно от модели.

- **«Каталожная» модель ≠ доступная модель.** Первый пин `gemini-2.5-flash` был
  выбран без ключа как «известный stable» — а живой demo-смок ответил
  `404: this model is no longer available to new users`: старые Flash-модели
  закрыты для новых API-пользователей, хотя в списке моделей всё ещё значатся.
  Актуальные ids пришлось снять с живого `GET /v1beta/models` под своим ключом
  (`gemini-3.6-flash`, `gemini-3.5-flash-lite`) и подтвердить одиночными
  `generateContent`. Мораль: model id без живой проверки под конкретным ключом —
  это гипотеза, а не пин; и это ровно тот класс глюков, ради которого демо
  гоняется по-настоящему, а не «по чек-листу».

## Отложенное / сознательные срезы

- **Env читается на импорте модуля агента** (`_env()` в обоих спайк-агентах).
  Для throwaway-спайка это норм, но тесты из-за этого вынуждены делать
  `monkeypatch.setenv` до отложенного импорта. Реальным planner/supplier
  (iter 1–3) стоит резолвить env лениво — не наследовать паттерн.
- **`_env()` продублирован в двух агентах намеренно:** caller не имеет права
  импортировать код remote-стороны (это демонстрируемая граница), а общий
  `shared/`-модуль запрещён конституцией. Три строчки дубля — цена честности
  границы.
- **`# tw-lite:`-маркеров в коде итерация не оставила** — резать было нечего:
  спайк и так минимален.
