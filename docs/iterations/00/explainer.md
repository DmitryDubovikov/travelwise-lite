# Итерация 00 — каркас + спайк A2A-канала

> 🎯 **Топология — продукт, приложение — фикстура.** Эта итерация не написала ни
> строчки travel-домена — и это осознанно: она зафиксировала стек и доказала, что
> A2A-канал ADK работает на пиннутых версиях, до того как на него встанет домен.

## Зачем это (продукт и ценность)

travelwise-lite — это планировщик поездок: пользователь пишет «4 days in Lisbon
in May», планировщик разбирает фразу, запрашивает офферы у независимого
поставщика перелётов и отелей и собирает из них готовый план. Ключевая
продуктовая идея — планировщик и поставщик принадлежат **разным владельцам**:
никто не импортирует чужой код, стороны общаются только по протоколу, как
турагентство общается с авиакомпанией через её сервис, а не через её исходники.
Ценность именно этой итерации — уверенность, что фундамент такой границы
работает: «рукопожатие» между двумя независимыми процессами прошло вживую ещё до
того, как появились сами перелёты и отели. Теперь домен строится на проверенном
канале, а не на надежде, что туториалы не соврали.

## 🧵 Что это дало резюме

Дифференциатор №1 — **A2A-граница**: умение выставить агента как отдельный
сетевой сервис и позвать его по открытому протоколу, а не импортом кода. Он
де-рискнут и частично демонстрируем уже сейчас, причём доказательства — не
слова, а артефакты: «визитку» агента (agent card) любой может скачать `curl`-ом
с фиксированного адреса, а в логе серверного процесса виден след протокольного
вызова (`GET /.well-known/agent-card.json 200` → `POST / 200`). Все термины из
этого абзаца разобраны в глоссарии ниже.

## TL;DR (простыми словами)

Было: пустой репозиторий с четырьмя документами о том, что мы собираемся делать.
Стало: рабочий питон-проект с зафиксированными версиями (`google-adk 2.5.0`,
`a2a-sdk 1.1.2`, `pydantic 2.13.4`, Python 3.13) и живым «спайком» — парой
одноразовых hello-агентов, которые доказали весь A2A-путь: один агент публикуется
как сервис одной строчкой `to_a2a()`, второй находит его по URL карты и зовёт по
протоколу. Плюс смок-тесты, которые никогда не зовут модель, и Makefile с тремя
командами. Спайк уже окупился: поймал недокументированную обязательную
зависимость (`a2a-sdk[http-server]`), без которой сервис просто не стартует.

## Глоссарий (если ADK и A2A видите впервые)

Целевой уровень читателя: вы знаете Python и, возможно, LangGraph/CrewAI из
сиблинг-проектов — но про ADK и A2A слышите впервые. Дальше по тексту эти
термины используются свободно.

- **Google ADK** (Agent Development Kit) — Python-фреймворк Google для сборки
  LLM-агентов, прямой конкурент LangGraph и CrewAI. Даёт готовые классы агентов
  и оркестраторов, CLI-тулинг и нативную интеграцию с Gemini; весь этот проект —
  освоение именно его.
- **Агент (`LlmAgent`, он же `Agent`)** — базовый кирпич ADK: объект «модель +
  текстовая инструкция (+ опционально инструменты)». Получает сообщение,
  прогоняет его через LLM по инструкции и возвращает ответ. `Agent` — просто
  короткий алиас класса `LlmAgent`.
- **`root_agent`** — конвенция обнаружения агентов: ADK считает питон-пакет
  агентом, если его `__init__.py` экспортирует переменную `root_agent`. Именно
  её ищут `adk run` и `adk web`, когда вы указываете им директорию.
- **`adk run` / `adk web`** — тулинг из коробки: `adk run <пакет>` — чат с
  агентом прямо в терминале; `adk web <директория>` — локальный веб-интерфейс
  со списком найденных агентов и трейсами вызовов. По роли это самописный
  Chainlit-UI плюс LangSmith-трейсы из мира сиблингов — только писать ничего
  не надо.
- **A2A (Agent2Agent)** — открытый протокол, по которому агенты **разных
  владельцев** зовут друг друга по сети как обычные веб-сервисы. Технически это
  HTTP: клиент сначала скачивает «визитку» агента (agent card), а потом шлёт
  вызовы в формате JSON-RPC. Реализация протокола живёт в отдельной библиотеке
  `a2a-sdk`; ADK подключает её как зависимость.
- **Agent card** — та самая «визитка»: JSON-документ, в котором сервис-агент
  объявляет имя, описание и скиллы (что он умеет делать по запросу). Аналог
  OpenAPI-спеки у REST-сервиса — контракт, по которому чужой клиент понимает,
  как с тобой разговаривать, не заглядывая в исходники.
- **Well-known-путь** — веб-конвенция: служебные файлы сервиса лежат по
  стандартному адресу, начинающемуся с `/.well-known/` (та же идея, что у
  `robots.txt` в корне сайта). Карта агента — всегда
  `/.well-known/agent-card.json`, поэтому клиенту достаточно знать хост и порт.
- **JSON-RPC** — простой формат «вызова функции по сети»: клиент шлёт POST с
  JSON вида `{"method": ..., "params": ...}`, сервер отвечает JSON-ом с
  результатом. Строчка `POST / 200 OK` в логе сервера — ровно такой вызов.
- **ASGI / Starlette / uvicorn** — стандартный стек питоньих веб-сервисов,
  который здесь достаётся «бесплатно»: ASGI — интерфейс между приложением и
  веб-сервером, Starlette — фреймворк, в котором приложение описано (на нём же
  построен FastAPI), uvicorn — сервер, который это приложение крутит. Функция
  `to_a2a()` возвращает готовое Starlette-приложение — остаётся отдать его
  uvicorn-у.
- **Gemini Flash / Flash-Lite** — быстрые дешёвые модели Google, доступные на
  бесплатном тарифе AI Studio (ключ выдаётся без карты и биллинга). Точные id
  зафиксированы в `.env`: `gemini-3.6-flash` и `gemini-3.5-flash-lite`.
- **Спайк** — не термин ADK, а приём разработки: минимальный одноразовый код,
  проверяющий рискованную гипотезу («A2A-канал на этих версиях вообще
  работает?») до того, как на ней строится основное.

## Что это за техника

Теперь, когда слова определены, — три вещи, которые итерация реально освоила.

**Agent card как контракт между владельцами.** Проблема: если агент — отдельный
сервис чужого владельца, клиенту нужно откуда-то узнать, что тот умеет и куда
слать запросы, не читая его исходников. В A2A эту роль играет карта: клиент
скачивает JSON с фиксированного well-known-адреса и дальше разговаривает по
нему — так же, как по OpenAPI-спеке с незнакомым REST-сервисом. Принципиально,
что в ADK карту не пишут руками: фреймворк генерирует её из определения агента
(имя, описание, скиллы), поэтому контракт не может разъехаться с кодом.

**`to_a2a()` — серверная половина границы.** Проблема: агент — это питоний
объект внутри процесса, а второй стороне нужен сетевой сервис. `to_a2a()` — это
функция «агент → готовый веб-сервис»: на входе обычный агент, на выходе
ASGI-приложение, где уже есть и карта на well-known-пути, и JSON-RPC-эндпоинт,
и исполнение агента при входящем вызове. В LangGraph-мире всё это была бы
самописная FastAPI-обвязка — роуты, сериализация состояния, документ-контракт
руками. Здесь — одна строка: `app = to_a2a(root_agent, port=8001)`.

**`RemoteA2aAgent` — клиентская половина.** Зеркальная проблема: из своего
агента позвать чужой сервис так, чтобы для остального кода он выглядел обычным
суб-агентом. `RemoteA2aAgent` объявляется одной конструкцией, которой передают
только URL карты; под капотом он сам резолвит карту и гоняет JSON-RPC. В
LangGraph это был бы самописный HTTP-клиент плюс код, превращающий ответ сервиса
обратно в состояние графа. Ключевое свойство границы: caller знает **только URL
карты** — ни импорта чужого кода, ни знания о том, что у той стороны внутри.

## ⚖️ Специфика ADK (vs LangGraph/CrewAI)

Межсервисная граница — главный контраст этой итерации:

| Задача | LangGraph/CrewAI | ADK |
|---|---|---|
| Выставить агента как сервис | Пишешь FastAPI-приложение: роуты, pydantic-модели запроса/ответа, запуск графа внутри хендлера | `app = to_a2a(root_agent, port=8001)` — карта и JSON-RPC-эндпоинт генерируются |
| Описать контракт для чужого клиента | Руками ведёшь OpenAPI/README и следишь, чтобы не разъехались с кодом | Карта выводится из определения агента автоматически |
| Позвать чужой сервис из своего агента | HTTP-клиент + маппинг ответа в state графа | `RemoteA2aAgent(name=..., agent_card="<url>")` — выглядит как обычный суб-агент |

Ложка дёгтя, без которой контраст был бы рекламой: вся A2A-поверхность ADK 2.5.0
(`to_a2a`, `RemoteA2aAgent`, конвертеры) помечена **EXPERIMENTAL** и при каждом
импорте честно предупреждает о возможных breaking changes — это одна из причин,
почему версии заморожены жёстко. Вторая шероховатость: заявленный extra
`google-adk[a2a]` не тянет всё нужное — серверные роуты падают без
`sse-starlette`, который приезжает только с `a2a-sdk[http-server]`. В LangGraph,
где обвязку пишешь сам, ты хотя бы видишь свои зависимости; здесь пришлось
узнать об этом от падающего сервера. И третье: у `to_a2a()` магии конвенций
больше, чем контроля — например, точный формат карты (см. learnings) фреймворк
решает за тебя.

## Поток данных

Триггер потока — команда `adk run spike/caller` (или сообщение агенту `caller` в
`adk web`): пользователь отправляет реплику hello-агенту, живущему в **другом
процессе**. Чтобы это было честной границей, а не вызовом функции, путь такой:
caller-у нужно узнать, *как* разговаривать с remote — поэтому он сначала
резолвит карту по `SUPPLIER_CARD_URL`; узнав эндпоинт, он шлёт JSON-RPC-вызов;
сервер исполняет своего агента (тот зовёт Gemini) и возвращает ответ по тому же
протоколу.

```
 Терминал А (сервис, :8001)                Терминал Б (caller)
 ┌─────────────────────────────┐           ┌──────────────────────────────┐
 │ make spike-remote           │           │ adk run spike/caller         │
 │                             │           │                              │
 │ to_a2a(root_agent) ────► карта          │ RemoteA2aAgent("hello_remote")│
 │   /.well-known/agent-card.json ◄──(1) GET── резолв карты по env-URL    │
 │                             │           │                              │
 │ POST /  ◄──────────────────────(2) JSON-RPC── "hi"                     │
 │   └─► hello_supplier (LlmAgent)         │                              │
 │         └─► Gemini Flash    │           │                              │
 │   ответ ───────────────────────(3)────► "[hello_remote]: Hi there!"   │
 └─────────────────────────────┘           └──────────────────────────────┘
```

| Компонент | Что делает | Что отдаёт |
|---|---|---|
| `spike/server.py` | Оборачивает hello-агента в `to_a2a()` и поднимает uvicorn на `SUPPLIER_PORT` | ASGI-сервис: карта + JSON-RPC-эндпоинт |
| `spike/remote_hello/agent.py` | `LlmAgent` на `gemini-3.6-flash`, отвечает одним приветствием | Текст приветствия |
| `spike/caller/agent.py` | `RemoteA2aAgent`, читает `SUPPLIER_CARD_URL` из env | Ответ remote-агента, как будто это его собственный |
| `tests/test_smoke.py` | Пять проверок пинов/карты/конструирования — без LLM и сети | Зелёный `make check` |

Честные оговорки: в этой итерации **нет** ни travel-домена (ни перелётов, ни
отелей, ни схем), ни оркестраторов (`SequentialAgent`/`ParallelAgent` — iter 1 и
3), ни настоящего supplier-а — оба агента одноразовые и умрут, когда их место
займут реальные (iter 1–3). Список того, чего «не происходит», и есть план
следующих итераций.

## Карта «где в коде»

1. **Пины стека — `pyproject.toml:6-13`.** Три зависимости зафиксированы точным
   `==`, потому что проект — сигнал fluency в *текущем* ADK, и дрейф версий
   убил бы воспроизводимость спайка. Extra `http-server` у `a2a-sdk` — не
   украшение, а обязательное условие старта сервера (см. learnings):

   ```toml
   dependencies = [
       # Pinned at the iter-0 spike (2026-07-22); no mid-project upgrades — see CLAUDE.md
       "google-adk[a2a]==2.5.0",
       # http-server extra: to_a2a()'s card/JSON-RPC routes need sse-starlette,
       # which plain a2a-sdk doesn't pull (found by the iter-0 spike)
       "a2a-sdk[http-server]==1.1.2",
       "pydantic==2.13.4",
   ]
   ```

2. **Серверная половина спайка — `spike/server.py:9-14`.** Модуль берёт
   hello-агента и одной строкой превращает его в A2A-сервис; порт приходит
   только из окружения (Makefile/`.env`), дефолта в коде нет — это шов
   конституции «никакого хардкода host/port»:

   ```python
   from spike.remote_hello.agent import _env, root_agent

   port = int(_env("SUPPLIER_PORT"))  # set by Makefile / .env — no default in code
   app = to_a2a(root_agent, port=port)

   if __name__ == "__main__":
       uvicorn.run(app, host="localhost", port=port)
   ```

3. **Hello-агент — `spike/remote_hello/agent.py:16-21`.** Обычный `LlmAgent` на
   пиннутом `gemini-3.6-flash` (id читается из env, в коде модели нет). Его
   единственная работа — ответить приветствием, чтобы round-trip было чем
   доказать:

   ```python
   root_agent = Agent(
       name="hello_supplier",
       model=_env("MODEL_FLASH"),
       description="Spike hello agent, reachable over A2A.",
       instruction="Reply with one short friendly greeting and nothing else.",
   )
   ```

4. **Клиентская половина — `spike/caller/agent.py:16-20`.** `RemoteA2aAgent`
   объявлен как `root_agent` пакета, поэтому его видит и `adk run`, и `adk web`.
   Всё, что он знает о второй стороне, — URL карты из `SUPPLIER_CARD_URL`;
   упавший env даёт внятный `RuntimeError`, а не загадочный `KeyError`:

   ```python
   root_agent = RemoteA2aAgent(
       name="hello_remote",
       agent_card=_env("SUPPLIER_CARD_URL"),
       description="Spike: calls the hello agent over A2A by card URL.",
   )
   ```

5. **Смок-тесты — `tests/test_smoke.py:28-64`.** Пять проверок, ни одна не
   ходит в сеть и не зовёт модель: установленные версии сверяются с пинами,
   *прочитанными из `pyproject.toml`* через `tomllib` (единственный источник
   истины); `SUPPLIER_CARD_URL` из `.env.example` обязан кончаться на константу
   ADK `AGENT_CARD_WELL_KNOWN_PATH`; model ids обязаны быть конкретными
   free-tier id (тест завалит и `-latest`, и `pro`); оба спайк-агента
   конструируются, а `to_a2a()` возвращает Starlette-приложение. `.env.example`
   парсится тем же `dotenv`, каким ADK читает `.env` в рантайме:

   ```python
   ENV = dotenv_values(ROOT / ".env.example")

   def test_model_ids_are_concrete_free_tier():
       for key in ("MODEL_FLASH", "MODEL_FLASH_LITE"):
           model = ENV[key]
           assert "latest" not in model, ...
           assert "pro" not in model, ...
   ```

6. **Каркас запуска — `Makefile:1-21` и `.env.example:1-16`.** Makefile
   подгружает `.env`, экспортирует его дочерним процессам и держит дефолты
   портов (`SUPPLIER_PORT ?= 8001`) — единственное разрешённое место для них
   вне `.env.example`. Три цели: `check` (pytest), `spike-remote` (A2A-сервис),
   `web` (`adk web` над `spike/`).

*Номера строк — ориентир на момент итерации 00; при последующих правках могут
сползти.*
